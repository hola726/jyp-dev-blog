# JYP Dev Blog - PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 비전
AI, Flutter, 모바일 개발 지식을 공유하고 개인 브랜딩을 구축하는 기술 블로그

### 목표
- **기술 기록/공유**: 학습한 기술과 경험을 체계적으로 정리하고 공유
- **개인 브랜딩**: 기술 전문성을 보여주는 온라인 포트폴리오
- **취업/이직 포트폴리오**: 기술 역량을 증명하는 콘텐츠 허브

### 핵심 가치
- 콘텐츠 중심의 깔끔한 디자인
- 빠른 로딩 속도 (Lighthouse 90+)
- SEO 최적화

---

## 2. 타겟 사용자 페르소나

### 페르소나 1: 동료 개발자
- **배경**: AI, Flutter, 모바일 개발 분야에서 일하는 개발자
- **목적**: 기술 참고, 문제 해결 방법 탐색
- **니즈**: 정확한 코드 예제, 실용적인 튜토리얼, 깊이 있는 기술 분석

### 페르소나 2: 넓은 독자층
- **배경**: 개발에 관심 있는 일반 독자, 학생
- **목적**: 학습, 정보 취득
- **니즈**: 이해하기 쉬운 설명, 단계별 가이드

### 페르소나 3: 채용 담당자
- **배경**: 기술 인재를 찾는 채용 담당자
- **목적**: 기술 역량 확인
- **니즈**: 프로필 정보, 기술 스택, 프로젝트 경험

## 3. 상세 기능 명세

### F1: 다크모드
- **요구사항**:
  - 라이트 / 다크 / 시스템 3가지 모드 지원
  - `next-themes` 라이브러리 사용
  - 전환 시 FOUC(Flash of Unstyled Content) 없음
  - 모든 컴포넌트(코드블록, 댓글 위젯 포함) 테마 연동
- **허용 기준**:
  - 페이지 새로고침 시 선택한 테마 유지
  - 시스템 테마 변경 시 자동 반영
  - 코드 블록 듀얼 테마 (라이트: github-light, 다크: one-dark-pro)

### F2: 포스트 목록 (무한 스크롤 + 태그 필터)
- **요구사항**:
  - 10개씩 로드, 스크롤 시 자동 추가 로드
  - PostCard 구성: 제목, 설명, 날짜, 태그, 읽기 시간
  - 최신순 정렬
  - 인라인 태그 뱃지 필터 (전체/태그별 토글)
- **허용 기준**:
  - 초기 로딩 시 10개 포스트 표시
  - 하단 도달 시 추가 10개 자동 로드
  - 로딩 인디케이터 표시
  - 모든 포스트 로드 완료 시 "더 이상 글이 없습니다" 표시
  - 태그 뱃지 클릭 시 해당 태그 포스트만 필터링, 재클릭 시 전체 복귀
  - 필터 변경 시 표시 개수 리셋

### F3: 포스트 상세
- **요구사항**:
  - 본문 중앙 배치, 우측 TOC(Table of Contents) 사이드바
  - 코드 하이라이팅 (라인 번호, 라인 하이라이팅, 복사 버튼)
  - 이전/다음 포스트 네비게이션
  - 하단 댓글, 관련 포스트 추천
- **허용 기준**:
  - TOC: IntersectionObserver로 활성 섹션 추적
  - 데스크톱: TOC 우측 sticky 고정
  - 모바일: TOC 아코디언 형태
  - 코드 복사 버튼 클릭 시 클립보드 복사 + 피드백

### F4: 태그/카테고리
- **요구사항**:
  - `/tags` 태그 디렉토리 (카드 그리드, 포스트 수 표시)
  - 태그 카드 클릭 시 `/tags/[tag]` 상세 페이지로 이동
  - `/tags/[tag]` 상세 페이지에서 `← 태그 목록` 링크로 `/tags` 복귀
- **허용 기준**:
  - `/tags` 페이지에 2~3열 카드 그리드로 전체 태그 표시 (글 수 내림차순)
  - `/tags/[tag]` 페이지에 해당 태그 포스트 목록 + 태그 목록 뒤로가기 링크
  - 포스트 카드에 태그 표시, 클릭 시 필터링

### F5: 검색
- **요구사항**:
  - Cmd+K(Mac) / Ctrl+K(Windows) 단축키로 모달 열기
  - 제목, 설명, 태그 기반 퍼지 검색 (Fuse.js)
  - 실시간 결과, 클릭/Enter로 이동
- **허용 기준**:
  - 검색 모달 열기/닫기 애니메이션
  - 타이핑 시 실시간 검색 결과 표시
  - 키보드 방향키로 결과 탐색, Enter로 이동
  - ESC로 모달 닫기

### F6: 댓글 (Giscus)
- **요구사항**:
  - GitHub Discussions 기반 댓글 시스템
  - 다크모드 자동 연동
  - lazy loading
- **허용 기준**:
  - 포스트 하단에 댓글 영역 표시
  - 테마 전환 시 댓글 위젯 테마도 자동 전환
  - 뷰포트 진입 시 로드 (IntersectionObserver)

### F7: RSS / SEO
- **요구사항**:
  - RSS 2.0 피드 (`/feed.xml`)
  - `sitemap.xml` 자동 생성
  - Open Graph, Twitter Card 메타태그
  - JSON-LD 구조화 데이터
- **허용 기준**:
  - `/feed.xml` 접근 시 유효한 RSS XML 반환
  - `/sitemap.xml` 접근 시 모든 페이지 URL 포함
  - 각 페이지에 적절한 OG 태그
  - 포스트 페이지에 Article JSON-LD

### F8: 보안 헤더
- **요구사항**:
  - `next.config.ts`에서 모든 경로에 보안 HTTP 헤더 적용
- **적용 헤더**:
  - `X-Content-Type-Options: nosniff` — MIME 스니핑 방지
  - `X-Frame-Options: DENY` — 클릭재킹 방지 (iframe 임베드 차단)
  - `Referrer-Policy: strict-origin-when-cross-origin` — 외부 사이트로의 URL 정보 유출 제한
  - `Permissions-Policy: camera=(), microphone=(), geolocation=()` — 불필요한 브라우저 API 차단

## 6. 기술 스택 및 아키텍처

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 16 (App Router) | SSG, React 생태계, 확장성 |
| 스타일링 | Tailwind CSS 4 | 빠른 개발, 다크모드, Next.js 공식 지원 |
| 콘텐츠 | MDX + Velite | Git 버전관리, 타입 안전 |
| 배포 | Vercel | Next.js 최적, 무료, 자동 배포 |
| 언어 | TypeScript | 타입 안전성 |

### 핵심 라이브러리
- `velite` - MDX 콘텐츠 관리
- `next-themes` - 다크모드
- `rehype-pretty-code` + `shiki` - 코드 하이라이팅
- `@giscus/react` - 댓글
- `fuse.js` - 클라이언트 사이드 검색
- `feed` - RSS 피드 생성
- `remark-gfm` - GitHub Flavored Markdown
- `rehype-slug` + `rehype-autolink-headings` - 헤딩 앵커
- `@tailwindcss/typography` - 산문 스타일
- `lucide-react` - 아이콘
- `date-fns` - 날짜 포맷
