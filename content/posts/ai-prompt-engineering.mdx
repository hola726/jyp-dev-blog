## 프롬프트 엔지니어링이란

프롬프트 엔지니어링은 AI 모델에서 원하는 결과를 얻기 위해 입력(프롬프트)을 설계하는 기술입니다.

## 기본 원칙

### 명확하게 작성하기

모호한 지시보다 구체적인 지시가 더 좋은 결과를 냅니다.

```text
❌ "코드 작성해줘"
✅ "Python으로 CSV 파일을 읽어서 날짜별로 매출을 집계하는 함수를 작성해줘"
```

### 컨텍스트 제공하기

AI에게 충분한 배경 정보를 제공하면 더 정확한 답변을 얻을 수 있습니다.

```text
"나는 Flutter 3.0을 사용하는 모바일 앱 개발자입니다.
현재 상태 관리로 Riverpod을 사용하고 있습니다.
비동기 API 호출의 에러 핸들링 패턴을 추천해주세요."
```

### 단계별로 사고하기

복잡한 문제는 단계별로 나누어 요청합니다.

```text
"다음 단계로 문제를 해결해줘:
1. 먼저 현재 코드의 문제점을 분석해줘
2. 가능한 해결 방법 3가지를 제시해줘
3. 각 방법의 장단점을 비교해줘
4. 최적의 방법으로 코드를 수정해줘"
```

## 고급 테크닉

### Few-shot 프롬프팅

예시를 제공하여 원하는 형식을 가르칩니다.

```python title="example.py"
# 입력-출력 예시를 포함한 프롬프트
prompt = """
입력: "2026-02-15" -> 출력: "2026년 2월 15일 (토)"
입력: "2026-01-01" -> 출력: "2026년 1월 1일 (목)"
입력: "2026-12-25" -> 출력: ?
"""
```

### Chain of Thought

AI에게 추론 과정을 보여달라고 요청합니다.

## 실전 활용 분야

- **코드 리뷰**: 버그 발견, 개선점 제안
- **문서 작성**: API 문서, README 생성
- **디버깅**: 에러 분석, 해결책 탐색
- **학습**: 개념 설명, 예제 생성
